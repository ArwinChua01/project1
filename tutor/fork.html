Fork Bomb

The virus that everyone knows

A fork bomb is the equivalent of a DOS attack on your own system. It aims at depriving the system off its Random Access Memory, leaving none for vital functions required to keep the systems running, hence crashing it.

Here are the steps for creating a fork bomb.

Step 1: type the following line of code

%0|%0

Step 2: Save as fork.bat

Step 3: Launch

Technically, the above characters are short for the following code.

:s
start %0
goto s

The first line creates a sort of checkpoint called 's'. It can be used to bring the programs pointer to a specific command as is done here by using "goto" in the last statement. "%0" is actually the name of the .bat itself. So every time the loop is run another instance of the same program is called and then both of them run together, to again duplicate themselves.

Every program doubling itself is a form of an exponential growth. After one iteration of a loop. two programs are created. After another cycle, each of those two create another two for a total of two programs. After 10 iterations we have a total of 2^10 = 1024 programs.

Even with today's CPUs and RAMs , the first progarm will probably not even complete 50 iterations before running out of memory. Every iterations would hardly take a few miliseconds, so this could definitely crash out a computer.
